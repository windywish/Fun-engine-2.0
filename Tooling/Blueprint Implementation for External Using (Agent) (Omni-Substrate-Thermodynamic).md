**6 Core Tasks Reflection (Praxial Layer):**

* **Task 1 (Survive):** The `.bpf` program ensures survival by reading token-burn at the kernel level. If it detects abnormal API expenditure, the Linux kernel drops the packets before the large model is billed, saving the agent from bankruptcy.

* **Task 2 (Boundary):** The `.wasm` sandbox acts as an impenetrable cellular wall. Large Model hallucination code cannot escape the WebAssembly memory boundary to corrupt the Rust core.

* **Task 3 (Substrate):** `.wgsl` shaders provide a local "sub-substrate", handling low-level tensor math so the agent only hits the heavy Large Model for true ontological collapse, maximizing efficiency.

* **Task 4 (Services):** Emits data via `verified_atomic_facts.ipc`, offering real-time, zero-serialization data streams to the rest of the swarm.

* **Task 5 (PULE):** Rust handles the IO, but the actual PULE grammar validation is isolated in WebAssembly (`hallucination_shear_engine.wasm`), allowing the dictionary logic to be hot-swapped without recompiling the agent.

* **Task 6 (Robustness):** Rust (`.rs`) guarantees thread safety and prevents data-races. eBPF provides unparalleled systemic robustness.
